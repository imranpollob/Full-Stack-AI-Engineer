# Week 5: Introduction to Machine Learning

Welcome to Week 5 of the Full-Stack AI Engineer bootcamp! We have officially crossed the threshold. You now know enough Python, Data Wrangling, and Mathematics to start training intelligent models.

This week is all about the fundamentals of **Machine Learning (ML)**. We will start with classical algorithms before moving towards Neural Networks in the coming weeks.

## What We'll Cover This Week

*   **Day 1: Machine Learning Basics.** We define the core terminology (Features, Targets, Overfitting, Underfitting) and introduce the three main types of ML: Supervised, Unsupervised, and Reinforcement Learning.
*   **Day 2: Supervised Learning & Regression.** We take the Linear Regression model from Math Week and officially use it as a predictive Machine Learning algorithm, utilizing Scikit-Learn to train and predict on continuous data.
*   **Day 3: Advanced Regression & Regularization.** What happens when your data isn't a straight line? We mathematically bend our lines using *Polynomial Regression*, and learn how to mathematically stop our models from overfitting using *Lasso* and *Ridge Regularization*.
*   **Day 4: Classification & Logistic Regression.** Not all AI predicts numbers. Often, it predicts categories (e.g., Spam or Not Spam). We introduce the math of classification using the famous Sigmoid Function.
*   **Day 5: Model Evaluation & Cross-Validation.** How do you *know* your model is good? We explore rigorous testing methodologies like K-Fold Cross-Validation, and define critical classification metrics like Precision, Recall, and the F1 Score.
*   **Day 6: k-Nearest Neighbors (k-NN).** We introduce a completely different machine learning algorithm that doesn't use algebraic slopes at all. Instead, it predicts answers purely by calculating the physical distance between data points!
*   **Day 7: The Supervised Learning Mini-Project.** We bring it all together. You will build an end-to-end Machine Learning pipeline on a real-world dataset, preprocessing the data, training multiple models, and mathematically proving which one is best.

## Why This Matters for AI
Deep Learning and LLMs (like ChatGPT) get all the hype, but those models are incredibly expensive and notoriously difficult to interpret.

In the real world, if you are predicting housing prices, detecting credit card fraud, or classifying customer churn, you do *not* use a billion-parameter neural network. You use the fast, cheap, and mathematically rigorous classical ML algorithms we are building this week.
